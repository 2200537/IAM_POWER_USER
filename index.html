MY-AWS-PROJECT-POWER_IAM_USER

  • Situation: Imagining a scenario where the IT team at Netflix need to give a senior engineer control over AWS resources without giving him full admin rights.

    Approach: In this situation, the IAM Power User role is used to give the engineer an access to manage resources like EC2 and S3 but can't access IAM users or roles. This ensures they have the access they need while keeping security controls in place..

Lab activities:

1. Creating an IAM Power User

2. Login the IAM Power user to the AWS Console

3. Test IAM Service Access

4. Test other Service Access

PROCEDURE:

A. CREATING A POWER IAM USER:

1. Login to AWS console homepage as a root user.

2. goto IAM Services and click on users

3. click-on create user

4. 5. Specify your user details

5. Tick to make your user an IAM user

6. Type in ‘power’ in the search tab under permission policies to Select your permission policy (PowerUserAccess)

7. Review and create user

11. Try to access other services on the console such as S3, EC2 etc.
